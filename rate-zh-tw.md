# 品質評分 — 繁體中文（台灣）

## 評分維度（4 × 2.5 = 10 分）

### D1：語言自然度（2.5 分）

| 分數 | 標準 |
|------|------|
| 2.5 | 完全自然，無法分辨是翻譯還是原創繁中內容 |
| 2.0 | 少數句子有翻譯痕跡，但不影響閱讀 |
| 1.5 | 多處可見英文句式影響 |
| 1.0 | 像機器翻譯加簡單修飾 |
| 0.5 | 明顯逐字翻譯，閱讀不舒服 |

檢查重點：
- 語序是否自然（非英文語序直譯）
- 是否使用台灣用語而非大陸用語（軟體/程式/網路/資料/部落格/伺服器/使用者/介面）
- 有無簡體中文用詞混入（"视频"→"影片"、"用户"→"使用者"、"博客"→"部落格"）
- 口語表達是否符合台灣習慣
- 科技術語英中混用是否自然

### D2：文化適配（2.5 分）

| 分數 | 標準 |
|------|------|
| 2.5 | 引用、案例和語調完美適配台灣讀者 |
| 2.0 | 大部分已適配，1-2 處仍偏美式 |
| 1.5 | 適配和未適配混合 |
| 1.0 | 很少文化適配，仍像美國內容翻譯 |
| 0.5 | 無任何文化適配 |

檢查重點：
- 比喻和類比是否對台灣讀者有意義
- 貨幣是否適當使用 NT$ 或美金
- 商業案例是否貼近台灣或亞太市場
- 語調是否適合台灣科技業專業人士

### D3：技術完整性（2.5 分）

| 分數 | 標準 |
|------|------|
| 2.5 | 所有元件、連結、圖片、格式完美保留 |
| 2.0 | 1 個小錯誤（多餘空格、格式不一致） |
| 1.5 | 2-3 個小技術錯誤 |
| 1.0 | 元件損壞或連結錯誤 |
| 0.5 | 多個元件損壞 |

檢查重點：
- `<SideCard>`、`<TryButton>`、`<BottomCard>` 完好且 title 已翻譯
- `<VideoPlayer>`、`<Table>` 的 content 未被修改
- 圖片 URL 保留
- Markdown 格式正確（標題、清單、連結、粗體/斜體）
- 內部連結有 `/zh-Hant/` 前綴

### D4：內容價值與 SEO（2.5 分）

| 分數 | 標準 |
|------|------|
| 2.5 | 內容價值等同或優於原文，關鍵字自然融入 |
| 2.0 | 內容價值好，關鍵字有但稍顯刻意 |
| 1.5 | 內容尚可但部分細節或衝擊力流失 |
| 1.0 | 資訊保留但缺乏吸引力 |
| 0.5 | 顯著的資訊價值流失 |

檢查重點：
- H1 > H2 > H3 結構保留
- 關鍵字適配台灣使用者搜尋習慣
- CTA 有效且自然
- 邏輯流暢度維持或改善
